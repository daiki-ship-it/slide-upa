# 台本形式の正本（slide-upa v1）

slide-upa の生成スキルが受け付ける台本の形式を定義する。

---

## 1. 基本構造

```markdown
# タイトル

## 第N章：章タイトル

### N-N. 節タイトル

（節の中身）
```

| 見出しレベル | 役割 | スライド |
|---|---|---|
| `#` | 講義タイトル | **表紙スライド 1枚** |
| `##` | 章（グループ） | **`###` 節がある章 → 章スライド 1枚**（アジェンダ転用・該当章を強調）。`###` がない章のみ **要点リスト 1枚** |
| `###` | 節 → **ここでスライドが切り替わる** | **1節 = 1枚** |

---

## 2. 節の中身

### 2.1 普通の段落

```markdown
### 1-1. 今日のゴール

まず今日のゴールです。具体的には3つあります。
一つ目は全体像を掴むこと。二つ目は...
```

→ Claude がキーポイントを 2〜4 つに自動要約して箇条書きスライドにする。

### 2.2 箇条書き

```markdown
### 1-1. 今日のゴール

まず今日のゴールです。

- 毎朝の進捗報告を自動化する全体像を掴む
- 自分の業務に置き換えるための4ステップを理解する
- 次回までにタスク一覧を1つ用意する
```

→ `- ` の直後のテキストをそのまま箇条書きスライドに使う（先頭に記号やアイコンは付けない）。  
→ `**語句**` は `<strong>語句</strong>` に変換する。

### 2.3 段落と箇条書きの混在

箇条書きが1つでもあれば、箇条書き行だけを使う（段落は `script` に残す）。

---

## 3. マーカー

| マーカー | 書く位置 | 意味 |
|---|---|---|
| `[goal]` | 節見出し直後の行（任意） | ⑥ ゴールスライドに切り替える（見出しに「ゴール」があれば省略可） |
| `[agenda]` | 節見出し直後の行（任意） | ⑦ アジェンダスライドに切り替える（見出しに「アジェンダ」があれば省略可） |
| `[no-chapter]` | `##` 見出し直後の行（任意） | 章スライドを作らない（`## オープニング` も同様に章スライドなし） |
| `[quote: セリフ]` | 節の本文中（どこでも可） | ③ 大きな一言スライドに切り替える |
| `[visual]` | 節見出し直後の行 | ④ 画像メインスライドに切り替える |
| `[image: 説明]` | `[visual]` の下 | 画像スロットに入れるキャラ or 画像の指示 |

### マーカーの例

```markdown
### 2-1. 自動報告の仕組み

今日お伝えするのは...

[quote: ポイントは「人が毎回書かなくていい」ことです]
```

```markdown
### 2-2. 全体の流れ
[visual]
[image: タスクを読む → 整理する → Slack に投稿する、3ステップの流れ]
[image: スマホで報告を確認するパニっくん]
```

### 3.1 `[image:]` の書き方（図解スライド）

画像は**文字なし**で、台本の内容が**一目で伝わる比喩**にする。スライド本文に語句があるので、画像内にラベルや見出しは不要（AI が文字を描くと崩れやすい）。

**推奨フォーマット**

```markdown
[image: 文字なし。伝えたいこと: （節の核心を1文）。比喩: （アイコン・配置・対比・流れの具体描写）]
```

| 要素 | 書くこと | 例 |
|---|---|---|
| **伝えたいこと** | 視聴者が覚えるべき1文（問題→解決も可） | 散らばったコピー→1つの正本に決める |
| **比喩** | 文字の代わりに**形・色・大きさ・矢印の太さ**で意味を出す | 左=薄く散らばった書類、右=大きな星付きフォルダ1つ |
| **スタイル** | 書かなくてよい | 白背景・ティール等は生成側が付与 |

**文字なしで伝えるコツ**

- **対比**: 散らばり vs 1か所、小 vs 大、薄い vs 太い、点線 vs 実線
- **アイコンの区別**: カバン/人物/旗/ノートなど形が違えば、ラベルなしでも4要素と分かる
- **視線誘導**: 左→右の流れ、最終状態（解決側）を最も目立たせる
- **1枚1メッセージ**: 複数の独立した概念を詰め込まない

**避ける書き方**

- 画像内に日本語ラベルや見出しを書く指示（崩れやすい・本文と重複）
- 配置・色・矢印の数だけ列挙（**何が問題で何が解決か**が見えない）
- `文字ラベルなし` だけ書いて比喩が薄い

**良い例**

```markdown
[image: 文字なし。伝えたいこと: 4種類の情報が1つにまとまりAIへ渡される。比喩: 4つの異なるアイコンが中央の大きな円に集約され、右端のAIへ太い矢印1本]
```

**キャラ画像**（図解ではなくキャラ差し込み）のときだけ、従来どおり感情付きで書く:

```markdown
[image: スマホで報告を確認するパニっくん]
```

## 4. スライド型の判定ルール

1. `# タイトル` → **① 表紙**
2. `## 章タイトル`（その章に `###` 節が1つ以上ある）→ **⑤ 章**（アジェンダと同系統。該当章の丸だけ塗る）。**例外:** 見出しが `オープニング`、または直後に `[no-chapter]` がある章は章スライドを作らず、節だけ生成する
3. `###` 節 に `[quote:]` がある → **③ 一言**（`[quote:]` が複数あっても最初の1つを使う）
4. `###` 節 に `[visual]` がある → **④ 画像**
5. `###` 節 の見出しに「ゴール」を含む、または `[goal]` がある → **⑥ ゴール**（淡いヘッダー + ①②③）
6. `###` 節 の見出しに「アジェンダ」を含む、または `[agenda]` がある → **⑦ アジェンダ**（淡いヘッダー + 縦タイムライン）
7. 上記以外の `###` 節 → **② 要点リスト**
8. `##` 章に `###` が1つもない → その `##` 見出しで **② 要点リスト** を1枚

---

## 5. エラーと警告

| 状況 | 動作 |
|---|---|
| `#` が1つもない | **エラー** → 生成中止。修正方法を表示 |
| `##` も `###` も1つもない | **エラー** → 生成中止 |
| `[visual]` はあるが `[image:]` が1つもない | **エラー** → 生成中止。`[image:]` を追加するよう案内 |
| `[visual]` はないが `[image:]` がある | **警告のみ**（生成は続行）。`[visual]` 追加を提案 |
| 会話形式行（`**ウパ博士：**`等）がある | **エラー** → 生成中止。一人語り形式への書き直しを案内 |

---

## 6. 禁止事項

- `**ウパ博士：**` / `**パニっくん：**` などの会話形式 → 使わない
- `## パニっくんとウパ博士の対話` などの対話セクション → 使わない
- 1つのファイルに複数の `#` タイトル → 使わない（最初の `#` だけを表紙にする）

---

## 7. 最小限の台本例

```markdown
# AIが毎朝タスク進捗を自動報告する仕組み

## 第1章：はじめに

### 1-1. 今日のゴール

まず今日のゴールです。

- 毎朝の進捗報告を自動化する全体像を掴む
- 自分の業務に置き換えるための4ステップを理解する
- 次回までにタスク一覧を1つ用意する

### 1-2. 一番覚えておくこと

[quote: ポイントは「人が毎回書かなくていい」ことです]

## 第2章：仕組み

### 2-1. 全体の流れ
[visual]
[image: タスクを読む → 整理する → Slack に投稿する、3ステップの流れ]
```
