# slide-upa 合意一覧（改訂版2）

**更新:** 2026-06-08  
**前提:** 台本は既存 `script-upa`（会話形式）とは**別物**。佐藤さんが**一人で話す**用の台本からスライドを生成する。ウパ博士・パニっくんは**台本の会話としては使わない**が、**スライドの見た目（世界観）** では使う。

---

## 1. 目的・スコープ

| 項目 | 内容 |
| --- | --- |
| **主用途** | MTG 中の画面共有（録画の背景資料） |
| **v1 で作るもの** | ① **スライドの見た目**（HTML） ② **確認・プレゼン用アプリ（studio）** ③ **生成スキル**（Cursor） |
| **やらないこと** | ポータル改修、台本作成スキル本体、振り返り図解ツール、既存 script-upa / diagram-upa の改修 |
| **ポータル** | 載せる必要があれば URL を手動共有。slide-upa v1 の範囲外 |

### 3つの成果物の役割

| 成果物 | 役割 | 誰が使う |
| --- | --- | --- |
| **生成スキル** | 台本から `output/` にスライド一式を生成 | 佐藤さん（カーソル） |
| **studio** | 全体の見た目確認・台本並列表示・手直し・MTG 操作 | 佐藤さん（localhost） |
| **audience.html** | MTG 共有用（スライドのみ） | クライアント（画面共有） |

---

## 2. 教育フローでの位置

```
① 台本作成スキル（新規・一人語り） → script.md 等
② slide-upa
     ├─ スキル（カーソル）→ output/{講義名}/
     └─ studio（localhost）→ 確認・MTG
③ 振り返り図解ツール（別途）         → 図解（会話形式でも可）
④ ポータル反映（必要時・手動）
```

- **script-upa（既存）** … 今回のフローでは使わない
- **diagram-upa（既存）** … 振り返り図解用。入力・ツールとも slide-upa とは別

## 3. プロジェクト構成

```
slide-upa/
├── AGREEMENT.md              # 本合意
├── assets/characters/        # キャラ PNG（単体完結の正本）
├── studio/                   # 確認・プレゼン用アプリ（v1: HTML+JS+Node）
├── output/
│   ├── _exemplar/            # スライド完成見本
│   └── {プロジェクト名}/
│       ├── audience.html     # MTG 共有用
│       ├── script.md         # 台本（コピー or 参照）
│       └── images/           # スライド用画像
├── references/
│   └── script-format.md      # 台本形式正本（予定）
└── .claude/skills/slide-upa/ # 生成スキル（予定）
```

| 項目 | 内容 |
| --- | --- |
| **名前** | `slide-upa` |
| **場所** | `src/slide-upa/`（`upahakase-visual-explainers` の配下ではない） |
| **キャラ画像** | `assets/characters/` に PNG を置く（**単体完結**）。初回は `upahakase-visual-explainers` から**コピー**。**図解リポ側は編集・移動・削除しない** |
| **公開 URL** | `slides-{プロジェクト名}-{YYYY-MM-DD}.surge.sh`（入口は `audience.html`） |
| **デザイン正本** | `output/_exemplar/` のスライド専用 HTML。色・フォントは diagram-upa を**参照** |

## 4. 入力（一人語り台本）

### 4.1 指定の仕方

- **生成時**: カーソルで台本のファイルパスを指定（例：「この `script.md` からスライド作って」）
- 台本の場所は固定しない（どこにあってもよい）

### 4.2 見出し構造（必須）

```markdown
# タイトル
## 第1章：章タイトル
### 1-1. 節見出し

（節の中身＝一人語り原稿）
```

| 見出し | 役割 |
| --- | --- |
| `#` | タイトル → **表紙スライド** |
| `##` | 章 |
| `###` | 節 → **基本はここでスライド切替**（v1 は ### 単位で試す） |

### 4.3 節の中身

- **普通の段落** と **箇条書き** の**両方**使ってよい
- **会話形式にしない**（`**パニっくん：**` / `**ウパ博士：**` は使わない）
- script-upa 専用の `## パニっくんとウパ博士の対話` 行も**不要**

### 4.4 形式チェック

- 上記ルールに合わない場合 → **スライドを生成せず、直し方を表示**して止める
- 事実確認（Web 検証）は**しない**（台本が正）

---

## 5. 台本 → スライドのルール

### 5.1 切替・枚数

| ルール | 内容 |
| --- | --- |
| 基本 | `###` 見出し **1つ = スライド 1枚** |
| 表紙 | 台本先頭の `# タイトル` から **1枚**（ウパ博士の絵） |
| 例外 | `###` が1つもない章は `##` 見出しで **1枚** |
| 枚数上限 | **なし**（台本の見出しとマーカーで決まる） |
| 自動ビジュアル判定 | **しない**（`[visual]` は手動指定のみ） |

### 5.2 レイアウト型（マーカー優先）

v1 の型は **4種類**。足りなければ試運用後に追加。

| 型 | いつ使うか |
| --- | --- |
| ① 表紙 | `# タイトル` |
| ② 要点リスト | マーカーなしの `###`（節内原稿を **自動要約**） |
| ③ 大きな一言 | `[quote: セリフ]` がある `###` |
| ④ 画像メイン | `[visual]` がある `###` |

---

## 6. マーカー

### 6.1 一覧

| マーカー | 書き方 | 意味 |
| --- | --- | --- |
| 画像多め | 見出し直後の行に `[visual]` | この節は④画像メイン型 |
| 画像内容 | `[image: 内容の説明]` | AI/キャラ画像の指示（`[visual]` の下） |
| 強調一言 | `[quote: 話す一言]` | ③大きな一言型（**話し手が強調するフレーズ**） |

### 6.2 例

```markdown
### 2-1. 自動報告の仕組み

今日お伝えするのは、毎朝 Slack に進捗が届く仕組みです。

- 前日のタスクを AI が読む
- 完了・未完了を整理する
- 決まった時間に投稿する

[quote: ポイントは「人が毎回書かなくていい」ことです]
```

```markdown
### 2-2. 全体の流れ
[visual]
[image: 3ステップの矢印図。Slack アイコンあり]
[image: スマホを見て安心するパニっくん]
```

### 6.3 エラー・警告

| 状況 | 動作 |
| --- | --- |
| `[visual]` あり、`[image:]` なし | **警告して生成しない**（修正後に再実行） |
| `[visual]` なし、`[image:]` あり | **無視＋警告**（生成は続行） |
| 同じ `###` 内に `[image:]` 複数行 | **1枚のスライドにまとめる** |
| `[quote:]` なし | ③は作らない（②要点リストのみ） |

## 7. スライドのデザイン・キャラクター

| 項目 | 内容 |
| --- | --- |
| 表紙 | ウパ博士の絵で世界観を出す |
| 各枚 | **右下**にページ番号、**左下**に小さなウパ博士アイコン |
| 枠線 | スライド上下にウパ博士**ブランドカラー**の線 |
| 著作権フッター | スライドには**載せない**（契約書でカバー） |
| まとめスライド | 特別指定なし。台本構成に従う |
| AI 画像 | `[image:]` の指示に従う。キャラ優先だが**細かいルールは試運用後に調整** |
| アスペクト比 | 16:9（画面共有向け） |
| デプロイ時 | `assets/characters/` から `output/{名}/images/` へ**実ファイルとしてコピー** |

## 8. studio（確認・プレゼン用アプリ）

### 8.1 目的

- **全体的な見た目**を確認する（主目的）
- **各スライドと台本を必ず同時に見える**ようにする
- **MTG の進行操作**（プレゼンター画面として機能）
- `presenter.html` は**作らない**（studio がその役割）

### 8.2 規模・技術

| 段階 | 内容 |
| --- | --- |
| **v1** | `studio/` に HTML + JS + 小さな Node サーバー。**localhost のみ** |
| **v2** | Next.js 本格アプリへ移行。Google スライド級機能（次枚プレビュー・タイマー等）も v2 |

### 8.3 画面レイアウト（ハイブリッド）

| モード | 内容 |
| --- | --- |
| **通常（3ペイン）** | 左＝サムネ一覧、中央＝スライド、右＝台本（**常時表示**） |
| **全体確認** | 縦スクロールで「スライド＋台本」セットを一覧表示 |

各スライドに**型ラベル**（表紙/要点/一言/画像）と**枚数カウント**を表示する。

### 8.4 プロジェクトの選び方

- 起動時は `output/` の**最新プロジェクトを自動表示**
- メニュー（ドロップダウン）で**過去の講義に切替**可能

### 8.5 手直し（v1）

| 種類 | v1 の扱い |
| --- | --- |
| **テキスト** | アプリ上で修正可。保存は**スライド側のみ**（台本は変えない） |
| **台本とのズレ** | **警告を表示** |
| **台本の構造変更** | カーソルで台本修正 → スキルで再生成 |
| **キャラ画像** | `assets/characters/` から**一覧で選び直し** |
| **`[image:]` 用画像** | **ファイルアップロード**で差し替え |
| **AI 画像の再生成** | カーソル（スキル）に任せる |

### 8.6 MTG 開始の動き

```
[studio のタブ]  ← あなたが操作（スライド＋台本が見える）

「MTG開始」を押すと…

[audience.html のタブ]  ← スライドのみ（Meet でこれを画面共有）
```

1. studio で確認・手直し
2. **「MTG開始」** → `audience.html` が**別タブ**で開く
3. studio で「次へ/前へ」→ audience タブも**同期**（BroadcastChannel）
4. Google Meet では **audience タブだけ**画面共有
5. studio タブは**そのまま残る**（いつでも台本を見ながら操作できる）

## 9. 全体フロー

```
【作成】
カーソルで台本パス指定 → 形式チェック → ワンショット生成 → output/{名}/

【確認・手直し】
studio を起動（localhost）→ 最新プロジェクトを表示
  → 3ペイン / 全体確認モードで見た目チェック
  → 必要ならテキスト・画像を手直し

【MTG】
studio で「MTG開始」→ audience.html を共有
  → studio だけ操作

【公開（v1）】
studio で「Surge公開」→ Surge にデプロイ → URL を返す（クリップボードコピー＋新しいタブ）
  ※ 未保存の変更がある場合は先に自動保存
  ※ 初回のみターミナルで `npx surge login` が必要
  ※ ポータルには URL を手動で載せる
```

## 10. v1 / v2 スコープ

### v1 に含める

- スライド完成見本（4型 + ウパ博士世界観）
- `audience.html`（共有用・同期受信）
- studio（3ペイン + 全体確認 + MTG開始 + 手直し + **Surge公開**）
- 生成スキル（Cursor）
- surge デプロイ（**studio からワンクリック**。スキルからも可）

### v2 以降

- studio の Next.js 化
- Google スライド級 UI（次枚プレビュー・タイマー等）
- `#slide-5` のような**特定枚リンク**
- ポータル `learnCard` への `slideHref` 追加（ポータル改修）
- 見出しレベル（`###` 固定か `##` も含むか）の**正式決定**
- スライド型の追加（Before/After、手順型など）
